# Système de Récompenses des Titres — RPSLS

**Date:** 2026-06-07  
**Usage:** Schéma complet du système de récompenses lié aux titres de victoire. Chaque titre débloqué confère des bonus cumulatifs qui impactent l'économie du joueur (XP, éclats, LP) et l'expérience de jeu (cosmétiques, effets visuels, avantages sociaux).  
**Philosophie:** Le titre n'est pas qu'un badge — c'est une **progression horizontale** qui récompense la variété de jeu sans créer de déséquilibre compétitif. Aucun bonus ne donne un avantage direct en match (pas de "dégâts +5%"). Tout est économique, cosmétique ou social.

## 2. Détail par rareté

### ⚪ COMMUN — Le socle, pas de bonus mais une fondation

| # | Perk | Description |
|---|------|-------------|
| C1 | **Déblocage du profil Titres** | Dès le 1er titre Commun débloqué, l'onglet "Titres" apparaît dans le Profil |
| C2 | **Compteur de collection** | Le nombre total de titres débloqués s'affiche sous le pseudo (ex: "🏆 12") |
| C3 | **Titre épinglable** | Le joueur peut épingler UN titre (même Commun) qui s'affiche partout |
| C4 | **Progression de palier** | 10 Communs → déverrouille un skin de cadre de profil "Bronze" |
| C5 | **30 Communs** → 🎖️ Badge "Vétéran des Arènes" + 500 XP (one-shot) |

**Pourquoi aucun bonus permanent?** Les Communs sont trop fréquents. Donner un bonus rendrait le système inflationniste. En revanche, les **paliers de collection** (10, 20, 30 titres) offrent des récompenses one-shot motivantes.

## 4. Impact sur l'économie joueur

### Flux de ressources

```
Match gagné
    │
    ├─→ XP de base (selon mode)
    │     ├─ +5%  si au moins 1 titre Rare actif
    │     ├─ +10% si au moins 1 titre Épique actif
    │     └─ +20% si au moins 1 titre Légendaire actif
    │
    ├─→ LP de base (online uniquement)
    │     └─ +5 LP si au moins 1 titre Légendaire actif
    │
    └─→ Éclats de base (selon mode)
          ├─ +1 si au moins 1 titre Épique débloqué
          └─ +3 supplémentaires si au moins 1 titre Légendaire débloqué
```

### Projection sur 100 victoires

| Profil joueur | XP gagné | Éclats gagnés | LP bonus |
|--------------|---------|---------------|----------|
| Nouveau (0 titre) | 5 000 XP | 0 éclats | 0 LP |
| Commun (10 titres) | 5 000 XP | 0 éclats | 0 LP |
| Rare actif (1+) | 5 250 XP (+250) | 0 éclats | 0 LP |
| Épique actif (1+) | 5 500 XP (+500) | 100 éclats | 0 LP |
| Légendaire actif (1+) | 6 000 XP (+1000) | 400 éclats | 500 LP |
| Légendaire (5+ titres) | 6 000 XP | 400 éclats | 500 LP |

**Conclusion:** Un joueur Légendaire gagne ~20% plus d'XP et ~400 éclats de plus qu'un nouveau sur 100 victoires. L'écart est significatif mais pas cassé — il récompense la fidélité sans rendre le jeu injouable pour les nouveaux.

## 6. Impact gameplay (indirect, jamais direct)

> **Règle d'or :** Aucun titre n'affecte les probabilités de victoire, les dégâts, la vitesse, ou tout autre paramètre de match. L'impact est strictement : économique, cosmétique, social.

| Domaine | Impact | Détail |
|---------|--------|--------|
| **Progression** | Accélérée | +5-20% XP et +1-4 éclats/win réduisent le temps de grind |
| **Boutique** | Plus d'achats | Plus d'éclats = plus de packs, plus de crafting |
| **Ladder** | Léger avantage | +5 LP/win en Légendaire aide à monter (mais ne compense pas une défaite) |
| **Collectionnisme** | Boucle de rétention | Les joueurs chassent les titres → jouent plus de modes → restent plus longtemps |
| **Statut social** | Reconnaissance | Les badges, auras et entrées custom signalent l'expérience sans exclure les nouveaux |

### Phase 1 — Stockage
```typescript

interface Player {
  // ... existant
  unlockedTitles: string[];          // IDs des titres débloqués ["titre_savant", "titre_vengeur"]
  activeTitle: string | null;        // Titre épinglé (peut être null)
  titleCollectionBonuses: {          // Paliers déjà réclamés
    commun10: boolean;
    commun20: boolean;
    commun30: boolean;
    rare5: boolean;
    rare10: boolean;
    rare21: boolean;
    epique5: boolean;
    epique15: boolean;
    legendaire3: boolean;
    legendaire10: boolean;
    legendaire21: boolean;
    collection5: boolean;
    collection10: boolean;
    // ... etc
  };
}
```

### Phase 2 — Évaluation des bonus actifs
```typescript
// engine/titles.ts — nouvelle fonction
export function getActiveTitleBonuses(player: Player): TitleBonuses {
  const titles = player.unlockedTitles.map(id => TITLES_REGISTRY[id]);
  const hasLegendaire = titles.some(t => t.tier === "legendaire");
  const hasEpique = titles.some(t => t.tier === "epique");
  const hasRare = titles.some(t => t.tier === "rare");

  return {
    xpMultiplier: hasLegendaire ? 1.20 : hasEpique ? 1.10 : hasRare ? 1.05 : 1.0,
    bonusEclats: (hasLegendaire ? 3 : 0) + (hasEpique ? 1 : 0),
    bonusLp: hasLegendaire ? 5 : 0,
    pseudoColor: hasLegendaire ? "gold" : hasEpique ? "purple" : hasRare ? "blue" : "white",
    hasAura: hasLegendaire,
    hasCustomEntrance: hasLegendaire,
  };
}
```

### Phase 3 — Application dans le flux de match
Dans `recordMatch()`, après avoir calculé xpDelta et lpDelta :
```typescript
const bonuses = getActiveTitleBonuses(player);
const finalXp = Math.round(xpDelta * bonuses.xpMultiplier);
const finalLp = lpDelta + (outcome === "win" ? bonuses.bonusLp : 0);
const finalEclats = eclatsDelta + (outcome === "win" ? bonuses.bonusEclats : 0);
```

### Phase 4 — UI des bonus
- **Écran de fin de match** : Afficher une ligne "Bonus Titre: +X XP (+Y%)" sous les gains
- **Profil → Onglet Titres** : Afficher les bonus actifs en haut de la collection
- **Profil → Onglet Titres** : Barre de progression vers le prochain palier
